Praxant — Logo Concepts

Every cell shows the mark on light and dark surfaces, with a 16-pixel favicon-scale rendering in the cell footer so you can judge how each holds up small.

Node Trinity

Three workers. One supervisor.
mono · light
mono · dark
color · light
color · dark

Description

Three filled circles arranged across the bottom (the workers), connected by curved cubic Bézier paths to a single larger node above-center (the supervisor). A thin "breathing" halo around the supervisor adds presence without bulk. The leftmost worker is slightly larger — a deliberate asymmetry that gives the mark visual energy.

Why for Praxant

This is the literal architecture of a Praxagent council, drawn. Workers report up; supervision is the focal element; the curved edges suggest data flowing rather than rigid wiring. It says exactly what Praxant is without metaphor.

Where to use

GitHub social card · documentation site header · primary mark on the landing page · talk-slide title slide.

How

Hand-written SVG, ~7 elements, viewBox="0 0 100 100". Animate the supervisor radius for a heartbeat effect on a hero section. Scales cleanly to 16×16 because the asymmetric weight reads even when small.

Strong points

  • Conceptually airtight — supervised multi-agent is the literal product story.
  • Asymmetric composition has real visual energy; never feels static.
  • The supervisor halo gives the focal node weight without making it heavier.

Drawbacks

  • Node-network marks are common in the agent-tools space; alone it doesn't signal Praxant specifically.
  • Tells the architecture but not the positioning (sovereignty, local hardware) — orthogonal to those values.
02

Hidden Ant Network

A network mark that is also, secretly, an ant.
mono · light
mono · dark
color · light
color · dark

Description

Three filled body circles arranged vertically (head, thorax, abdomen) connected by thin petiole lines, with jointed leg polylines splaying from the thorax and elbow-jointed antennae from the head. The abdomen is the largest and the focal element. From a distance: an abstract three-node mark with delicate connections. Up close: an ant emerges from the geometry.

Why for Praxant

The only concept where the mark is uniquely Praxant's — the name "Prax-ant" is encoded directly into the geometry, but with enough abstraction that the mark doesn't read as a cartoon. The "jazz musician quoting another song" effect: insiders catch it; everyone else just sees a refined network mark with delicate appendages.

Where to use

Strongest as the primary brand mark — the one you put on stickers, the one you tattoo onto the loading screen. The dual reading is the point and it holds up at every scale.

How

17 elements: 3 body circles, 2 petioles, 6 jointed legs (each a 3-point polyline), 2 jointed antennae. Animate the antennae with a subtle rotate for a "twitch" on hover.

Strong points

  • Distinctive. Cannot be mistaken for any other agent-tooling brand.
  • Jointed legs and antennae give it anatomical refinement — not a kid's drawing.
  • Rewards attention without demanding it; the pun is a discovery, not a punchline.

Drawbacks

  • The pun is invisible to anyone unfamiliar with the project name etymology.
  • Structurally tied to the literal letters — gets weaker if "Praxant" rebrands.
  • At extreme small sizes (16×16), legs and antennae blur into the body.

Council Convergence

Many voices. One synthesis.
mono · light
mono · dark
color · light
color · dark

Description

Five council voices positioned asymmetrically around the perimeter, each connected by a curved cubic Bézier to a single larger center node (the synthesis). The lead voice (top-left) is slightly larger and connected by a noticeably heavier edge — communicating that not all voices carry equal weight in the deliberation.

Why for Praxant

This is the multi-model review council (project SP-7) drawn as geometry. The varied edge weights are the key refinement: a council isn't five identical voices summed equally — there are lead reviewers, supporting voices, dissenting opinions. The mark conveys that hierarchy without text.

Where to use

Marketing materials about the council feature; the SP-7 documentation page; explainer diagrams. Could work as the primary mark if council-as-differentiator is the chosen positioning.

How

11 elements: 5 outer dots + 5 weighted edges + 1 synthesis node. Animate by sequentially fading each edge in to dramatize "voices arriving at consensus" in a hero animation.

Strong points

  • Tells the most differentiated part of the Praxant story (multi-model council is unique).
  • Varied edge weights add narrative depth — clear lead voice, supporting voices.
  • Center-focal composition feels like an institution, not a startup.

Drawbacks

  • Without context, could read as a hub-and-spoke topology — exactly the opposite of council.
  • Five-around-one is a common visualization for "platform + integrations."
04

Praxant Wordmark

Just the name. With one quiet detail.
mono · light
mono · dark
color · light
color · dark

Description

Lowercase "praxant" set in a clean geometric sans, with a custom "x" — but instead of two simple crossed strokes, the X is constructed as four edges meeting at a node. Each stroke terminates short of the center, and a small filled circle sits at the intersection. The X is now visually a tiny network diagram embedded in the typography.

Why for Praxant

Wordmarks age the best of any logo type. They never look dated, they don't compete with the product UI, and they don't need explanation. The four-edges-meeting-at-a-node X is a graphical pun: every time you read the word, you literally see an agent network. It's the same network metaphor as the symbolic marks, expressed inside the typography itself.

Where to use

Documentation site header · README badges · presentation slide footers · business cards. Pairs with a pictorial mark (e.g., #02 or #08) for a vertical lockup.

How

Wider viewBox (0 0 220 64) since wordmarks aren't square. Uses system fonts via the cascade so it renders identically on macOS/iOS. Before shipping: outline the text via inkscape --export-text-to-path for absolute font consistency across platforms.

Strong points

  • Most legible at any size; the X-as-network detail is the only ornament.
  • Works in any color, on any background, in any context.
  • Minimal symbolic baggage — won't constrain future positioning shifts.

Drawbacks

  • No standalone mark for places that need a symbol only — must be paired.
  • System font fallback isn't pixel-identical across platforms unless outlined.
05

Hex Colony

Seven cells. One queen.
mono · light
mono · dark
color · light
color · dark

Description

Seven hexagons in a tight honeycomb cluster — five outer cells outlined at standard weight, the center cell outlined at slightly heavier weight (the coordinator), and the northeast cell filled solid (the queen / focal). The center cell's heavier stroke creates a subtle three-tier hierarchy: queen → coordinator → workers.

Why for Praxant

Honeycomb is the natural visual idiom for distributed-yet-coordinated, which is exactly the praxagent fleet model. The asymmetric queen position (NE rather than center) gives the mark dynamic energy. The subtle stroke hierarchy on the center hex hints that the cells aren't all peer — there's structure underneath.

Where to use

Documentation pages about the praxagent fleet model · cluster/orchestration diagrams · tiled background pattern. Less suited as a primary mark because hexagons are over-grazed in dev tooling.

How

Uses <defs> + <use> so the hex shape is defined once. Easy to recolor individual cells if you want a "fleet status" interactive widget where each hex represents a live agent.

Strong points

  • Pattern hints at scalability — the colony could keep growing.
  • Subtle stroke hierarchy on the center cell adds depth without complexity.
  • Excellent for product UI elements that need a "node grid" visual.

Drawbacks

  • Hexagon fatigue. Honeycomb is one of the most overused dev-tool motifs.
  • In monochrome, the queen distinction (filled vs outlined) is the only differentiator.

Ant Glyph

The ant, said out loud.
mono · light
mono · dark
color · light
color · dark

Description

An unambiguous but anatomically refined ant: head, thorax, and abdomen as filled circles linked by thin petiole connectors, with six jointed legs (femur + tibia segments) splaying from the thorax and elbow-jointed antennae sweeping forward from the head. The abdomen is the largest body segment and the focal element. Where #02 hides the ant inside an abstract network, #06 commits.

Why for Praxant

This is the most direct expression of the name. Ants are an almost-perfect metaphor for what Praxant does: many small autonomous workers under collective coordination, doing real labor without spectacle. The jointed legs and proper proportions make this ant look refined, not mascot-y.

Where to use

Strong as a community/cultural mark — Discord avatar, sticker, contributor t-shirt, mascot in animations. Probably not the primary corporate mark unless the brand voice leans warm.

How

17 elements: body circles + petioles + jointed legs (each a 3-point polyline) + jointed antennae. Could be animated with a subtle leg-cycle for delightful loading-state usage. Use round line caps and joins throughout to keep the joints elegant.

Strong points

  • Memorable. The pun is impossible to miss.
  • Jointed legs avoid the "kid's drawing" feeling of straight-line ants.
  • Strong personality without being childish.

Drawbacks

  • At favicon size, legs and antennae disappear into the body.
  • Some enterprise audiences read "literal animal" as not-serious-enough.

Orbital Agents

Parallelism, drawn in perspective.
mono · light
mono · dark
color · light
color · dark

Description

Three thin tilted ellipses representing orbital paths viewed from an angle (not concentric circles), with four small worker dots positioned on those orbits. A central orchestrator with a thin halo sits at the focal point. The tilt is the key refinement: tilted ellipses read as a 3D system in perspective, breaking the "atomic diagram" association of nested circles.

Why for Praxant

Visualizes the orchestrator-and-workers model with the addition of parallelism — workers on differently-tilted orbits implies they're doing different things at different cadences, which is how a praxagent fleet operates.

Where to use

Marketing pages about scheduling/concurrency · the SP-1 (orchestrator spine) documentation · a dashboard widget showing live agent activity.

How

Three rotated ellipses + four worker dots + halo + center node. Animate by slowly rotating each worker around its orbit at different speeds — actively communicates the parallelism story.

Strong points

  • Tilted ellipses break the "atom diagram" cliché — looks like a system, not chemistry.
  • The orbit paths make the "scheduling" story visible without text.
  • Animates beautifully — orbits at different speeds = visible concurrency.

Drawbacks

  • Even with tilted orbits, retains some risk of physics/astronomy reading.
  • Most complex mark in the set; worst at very small sizes.

Praxis P-Mark

The letter, with a pivot.
mono · light
mono · dark
color · light
color · dark

Description

A geometric uppercase "P" — a heavier vertical stem with a generous curved bowl — with a filled circle at the inner corner where the bowl rejoins the stem. The pivot circle is sized just larger than the stroke width, so it reads as a deliberate "knot" or "anchor" where the two strokes meet, not an applique.

Why for Praxant

Letterforms with one deliberate detail are the workhorse logos of serious technical brands (Postgres, Vercel, Supabase). The pivot node both completes the letter visually and ties back to the node-network language used everywhere else in the system. Stem stroke is thicker (5px vs the previous 4px) for more visual mass at small sizes.

Where to use

App icon · favicon · social avatar — anywhere you need a square mark that reads instantly. Pairs naturally with the wordmark (#04) as a vertical lockup for documentation.

How

Three drawn elements: stem line, bowl path, pivot circle. stroke-linecap="round" on stem and bowl for soft termination. Could rotate the entire mark 90° for a horizontal lockup variant.

Strong points

  • Compact. Best variant for 16×16 / 32×32 / app-icon use.
  • Single-letter monograms are timeless when geometric.
  • The pivot detail gives personality without requiring explanation.

Drawbacks

  • Single-letter monograms are an extremely crowded space.
  • Doesn't carry any of the multi-agent symbolism — purely an identity mark.

Forge Anvil

Praxant builds things. With heat.
mono · light
mono · dark
color · light
color · dark

Description

A more anatomically correct anvil silhouette — tapered horn on the left, working face, narrow waist (tapering trapezoid neck), and a wide planted base with rounded corners. Above the working face: three short dynamic spark lines radiating at varying angles plus two small spark dots, suggesting active hammer strikes rather than static decoration.

Why for Praxant

Echoes Forgejo (the v1 VCS adapter) and the broader self-hosted/homelab cultural aesthetic. "Praxis" shares roots with practice/craft/making; the anvil literalizes that. Anchors the brand in the world of people who build their own infrastructure rather than the world of people who consume managed services.

Where to use

Probably not as the primary mark — too literal — but excellent as a community badge, contributor recognition, or as the icon for a specific feature like a "Forge" build subsystem.

How

Three filled shapes for the anvil body (top + waist + base), three spark lines + two dots above. The dynamic sparks are the focal element in color (copper).

Strong points

  • Strong cultural signaling for the self-hosted/homelab audience.
  • Refined anvil silhouette (waist taper + planted base) — not generic blacksmith icon.
  • Conveys the craft angle of the brand more directly than any other variant.

Drawbacks

  • Visually adjacent to Forgejo's own identity — risks blurring platform vs adapter.
  • Says nothing specifically about agents or orchestration; works for any tool.
  • Risks reading as "blacksmith hobbyist" rather than serious infrastructure.

Container Bracket

Three agents, in a sandbox.
mono · light
mono · dark
color · light
color · dark

Description

Four L-shaped corner brackets — one in each corner of the canvas — implying a full container/sandbox boundary. Inside, a three-node mini-network: two worker nodes above with curved edges converging on a single supervisor below (focal). Brackets use square line caps for crisp structural feel; the inner network uses round caps for organic contrast.

Why for Praxant

Containerization is core to Praxant — every praxagent runs in an isolated container. The four-corner brackets visually represent that boundary; the network inside represents the supervised agents within. Tells the whole architecture story: agents (nodes), supervision (edges), isolation (brackets), in one mark.

Where to use

Architecture diagrams · the SP-5 (container runtime) documentation header · security-positioning pages where the isolation story matters.

How

Eight elements: 4 corner brackets (each a 3-point polyline) + 3 network nodes + 2 connecting paths. The contrast of linecap="square" on brackets vs linecap="round" on edges is intentional — it visually separates "structure" from "agents."

Strong points

  • Tells the most complete Praxant story: agents + supervision + isolation.
  • Four-corner brackets read clearly as container — better than two diagonal corners.
  • Strong rectangular silhouette — works well in horizontal lockups.

Drawbacks

  • Brackets can read as a code-syntax cliché.
  • "Busy" relative to the others (8 elements vs 3 for the wordmark).
  • The containerization reading isn't obvious without context.

Recommendation

If forced to pick one as the primary brand mark, lead with #02 Hidden Ant Network — it's the only concept that is uniquely Praxant's (the others could fit any agent-orchestration startup) — paired with #04 Wordmark as the horizontal lockup. #08 Praxis P-Mark completes the system as the favicon and app icon.

For a more conservative direction (avoiding any "ant" branding risk), #01 Node Trinity + #04 Wordmark + #08 P-Mark is a fully professional system that says "supervised multi-agent" without the pun.

Don't ship more than one mark per use case — pick the system, delete the rest, move on.
