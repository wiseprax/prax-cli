# Praxant.com Domain Acquisition — Step-by-Step Guide

**Purpose:** Pursue acquisition of `praxant.com` from its current registrant (registered 2026-01-31 via Gandi, GB-based, contact email `fa15ec50ebc6e61d282dba520cb8e3ab-58145410@contact.gandi.net`).

**Status:** Optional. The Praxant project will succeed without `.com` — `praxant.ai` is the canonical domain. This guide exists so you have a clear path if you decide the `.com` is worth pursuing.

**Estimated cost range:** $0 (no reply) → $500–$2,000 (squatter sale) → $5,000–$50,000+ (anchored / commercial). Walk away above $2,000 unless you have a specific business reason.

**Estimated time:** 1–4 weeks elapsed (most is waiting for the seller to respond).

## Why a multi-TLD strategy (and where each domain fits)

Praxant's domain strategy is **deliberately multi-TLD**. Each top-level domain serves a distinct purpose; together they form a coherent identity that no single TLD can provide.

### praxant.ai — Canonical domain (primary)

**Status:** Already reserved (2026-04-27).
**Cost:** ~$80–$160/year (Anguilla raised prices in 2024–2025).
**Role:** The official, day-to-day, marketing-facing identity of the project.

**Why it matters:**

| Reason | Detail |
|---|---|
| **Industry-standard signal for AI in 2026** | The `.ai` TLD has become the de facto namespace for AI products (Claude.ai, perplexity.ai, character.ai, mistral.ai). Using it puts Praxant immediately in the right mental category for developers searching for AI tooling. |
| **Premium positioning** | Despite the cost, `.ai` reads as "modern, well-funded, serious AI project." This matters for adoption — developers trust `.ai` projects to be active, current, and AI-focused. |
| **Globally recognized** | No regional connotation (despite being Anguilla's ccTLD); developers in every region understand the AI signal. |
| **Aligns with the GitHub org** | `praxant-ai` GitHub org + `praxant.ai` domain = visually coherent identity across surfaces. |
| **AI-tool retrieval** | When LLMs are asked "what is Praxant?" or "self-hosted AI agent platforms," the `.ai` TLD reinforces the categorization. |

**Tradeoffs accepted:**
- Higher annual cost than other TLDs (~5–10× the price of `.com`)
- Anguilla ccTLD — small political risk if Anguilla changes domain policy
- Annual-only renewal (no multi-year locks)

**Bottom line:** The flagship URL. All marketing, docs, package install instructions, and external references point here.

### praxant.io — Tech-credible secondary (recommended to acquire)

**Status:** Recommended for registration if available (~$30–$60/year).
**Role:** Developer-facing alias / fallback / redirect target.

**Why it matters:**

| Reason | Detail |
|---|---|
| **Tech-startup standard since ~2014** | The `.io` TLD has been the developer-tools default for over a decade (temporal.io, netlify.io, vercel.io originally). Reads as "this is a modern tech project" instantly. |
| **Habit-driven discovery** | When developers can't recall a project's full TLD, many type `.io` first by reflex. Owning it captures that traffic. |
| **Defensive against typo-squatters** | If someone hears about Praxant but doesn't know the TLD, `praxant.io` is one of the first guesses. Better that we own it than someone else hosts a fake. |
| **Often available when `.com` is taken** | The `.io` namespace is large enough that distinctive coined names are usually still acquirable. |

**Tradeoffs accepted:**
- Slightly dated feel compared to peak `.io` years (2015–2020)
- ccTLD politics (BIOT — British Indian Ocean Territory — is a contested jurisdiction; future status uncertain)
- Less differentiated than `.ai` for AI-specific positioning

**Bottom line:** Redirect or alias. `praxant.io` should resolve to `praxant.ai` (or vice versa). Owning both removes ambiguity in developer discovery.

### praxant.org — Open-source / sovereign signal (recommended to acquire)

**Status:** Recommended for registration (~$10–$15/year — cheapest of the four).
**Role:** Open-source positioning + sovereign-tech alignment + foundation-readiness.

**Why it matters:**

| Reason | Detail |
|---|---|
| **Universal "non-profit / OSS" signal** | The `.org` TLD has been the canonical non-profit and open-source namespace since the 1980s. For a project positioned as "sovereign, portable, open-source infrastructure," `.org` reinforces the values. |
| **Sovereign-friendly governance** | `.org` is managed by the Public Interest Registry, a non-profit. No corporate or national government control. Aligns with sovereign-tech positioning that distrusts US-cloud SaaS dependencies. |
| **Foundation-ready** | If Praxant later joins a foundation (Open Collective, Apache, Linux Foundation, NLnet, Sovereign Tech Fund), `.org` is the expected canonical TLD for foundation-hosted projects. Owning it from day one removes friction. |
| **Cheap and stable** | $10–15/year, multi-year renewal allowed, no surprise pricing changes (unlike `.io` and `.ai`). |
| **Long-term defensible** | Has the most mature trademark and dispute resolution infrastructure of any TLD. UDRP claims on `.org` are well-precedented. |

**Tradeoffs accepted:**
- Less "modern startup" feel than `.ai` or `.io`
- Some users may mistakenly assume Praxant is a registered non-profit (it isn't, at least initially)

**Bottom line:** Permanent home for documentation, foundation page, contributor guide if Praxant ever moves to foundation governance. Worth owning even if it just redirects to `praxant.ai` for now.

### praxant.com — Defensive future-proofing (the subject of this document)

**Status:** Currently held by someone else (registered 2026-01-31, GB-based, identity unknown). This document covers the acquisition attempt.
**Cost (if achievable):** $200–$2,000 from squatter, walk away above that.
**Role:** Brand protection, typo defense, professional email, long-term insurance.

**Why we want it (even though `praxant.ai` is the canonical):**

| Reason | Detail |
|---|---|
| **Universal default behavior** | When users hear "praxant" and don't know the TLD, they type `praxant.com` first. Browsers, mobile keyboards, and address-bar autocomplete all default to `.com`. Owning it captures lost traffic. |
| **Brand protection** | If someone else operates under `praxant.com`, they capture name-recognition value Praxant builds. Worse, they could host malicious content (phishing, malware) and damage Praxant's reputation by association. |
| **Typo-squatting defense** | Combined with `praxant.ai` / `.io` / `.org`, owning `.com` closes the "easy mistake" surface where users land somewhere unintended. |
| **Professional email** | `@praxant.com` reads more professional than `@praxant.ai` for non-AI contexts (legal correspondence, partnership outreach, vendor communications). Useful for the maintainer's official project email. |
| **Trademark strength** | USPTO and similar trademark offices give weight to whether the registrant owns the matching `.com`. Having it strengthens future trademark applications and defenses. |
| **Resale value** | If Praxant ever needs to sunset, `.com` is the only TLD with a meaningful resale market. Other TLDs have near-zero resale value. Insurance against project failure. |
| **Acquisition target prevention** | If a competitor or hostile entity acquires `praxant.com` later, they can pressure or confuse Praxant's user base. Owning it removes this attack vector permanently. |
| **Search engine canonicalization** | Search engines historically give slight preference to `.com` for "the official site of X." Owning both `.com` and `.ai` lets you set the canonical signal cleanly via redirects. |

**Tradeoffs accepted:**
- Acquisition cost (anywhere from $0 to several thousand dollars)
- Time investment in negotiation (1–4 weeks)
- May not be achievable at all (squatter refuses; or it's a stealth entity that won't sell)

**Bottom line:** Defensive purchase. Not required for the project to succeed, but valuable insurance against future brand confusion, trademark disputes, and traffic leakage. **The acquisition pursued in this document is specifically about future-proofing — not about needing `.com` for daily use.**

## Step 0 — Pre-flight: verify the situation before sending anything

Before contacting the registrant, confirm what `praxant.com` actually is. This determines your strategy.

### 0.1 Check what loads at praxant.com

```bash
# DNS resolution
dig praxant.com +short

# HTTP response
curl -sIL praxant.com

# What renders in a browser
# (Open praxant.com in an incognito window)
```

**Interpret the result:**

| Result | What it means | Your strategy |
|---|---|---|
| Gandi parking page ("This domain is parked") | Squatter or forgotten registration | High chance of cheap sale ($200–$2k); proceed |
| "Coming soon" / placeholder splash | Stealth entity building something | Higher risk — they may not sell at any price |
| Real product / company website | Active business | Evaluate their domain (AI? consulting? unrelated?) — if competitor, walk away; if unrelated, may sell |
| DNS error / no DNS records / connection refused | Squatter who hasn't even configured DNS | Strong sale candidate; very likely cheap |
| Redirect to another domain | Owned by larger entity using it for redirects | Unlikely to sell; walk away |

### 0.2 Check the registrant's broader footprint

```bash
# Reverse-search for other domains owned by the same registrant
# (Use a service like ViewDNS reverse whois, DomainTools, or WhoisXML)
# Look for: do they own many speculative domains? = squatter pattern
```

**If you find** the same registrant owns 50+ domains in random categories → confirmed squatter → cheap sale likely.
**If you find** they own a small portfolio of related/branded domains → a real business with brand strategy → less likely to sell cheaply.

### 0.3 Check trademark filings on "Praxant"

Quick search at https://tmsearch.uspto.gov and https://www.tmdn.org/tmview/

If "Praxant" has an active trademark filing in software/AI classes (009 or 042) → there's a real brand attempting to protect the name. **This changes your project name decision more than the domain question.** Pause and reconsider before proceeding with the acquisition.

If no trademark filings → no immediate conflict; proceed.

## Step 1 — Send the inquiry email

### 1.1 Set up an anonymous-feeling email account

**Don't use your work email or any address tied to a real company.** Squatters Google the email domain before quoting; a corporate email = higher quote.

Create a fresh account at one of:
- Gmail (most common, looks generic)
- ProtonMail (privacy-focused but still reads as personal)
- Tutanota (similar to Proton)

Use a generic-sounding name: e.g., `wellington.research@gmail.com`, not `wellington@yourcompany.com`.

### 1.2 Use this exact email template

**Subject line:**
```
Inquiry about praxant.com
```

**Body:**
```
Hi,

I came across praxant.com while looking at available domains
for a personal project. I noticed it's registered but doesn't
seem to be in active use.

Would you consider selling it? If so, what price would you ask?

No pressure either way — just exploring options.

Thanks,
Wellington
```

**Send to:**
```
fa15ec50ebc6e61d282dba520cb8e3ab-58145410@contact.gandi.net
```

### 1.3 What NOT to include in the email

| Don't say | Why |
|---|---|
| The name "Praxant" as a project | Telegraphs you have a project committed to this exact name; they will demand more |
| Mention of `praxant.ai` or your GitHub org | Same problem — proves you're invested |
| AI / agent / startup / company | Raises perceived commercial value of the name |
| "I really need this" or any urgency | Removes your bargaining power |
| An offered price | **Always let them quote first.** Anchoring with your own number is a beginner mistake |
| Your last name, employer, real signature | They will Google you; lower-profile = lower quote |
| Threats of UDRP / trademark / legal action | Adversarial tone kills the deal; only use if you actually plan to file UDRP (and you almost certainly don't) |

### 1.4 Send and wait

Send the email. Then **stop thinking about it for 1-2 weeks.** Don't follow up. Don't second-guess. The waiting period is part of the negotiation — eagerness raises the price.

Set a calendar reminder for **2 weeks from send date** to revisit.

## Step 2 — Interpret the response (or lack thereof)

### 2.1 No reply within 2 weeks

**Most likely outcome.** Many squatters don't actively monitor the contact email; many stealth entities prefer not to engage.

**Action:** Walk away. Do not send a follow-up. Move on with `praxant.ai` as your canonical domain. You can re-attempt the inquiry in 6-12 months — circumstances change (squatters let domains expire; stealth projects abandon).

### 2.2 They reply with a quote

Use this decision matrix:

| Quote range | What it likely means | Recommended response |
|---|---|---|
| **Under $200** | They forgot they registered it, want only registration costs back | Accept immediately. Move to Step 3. |
| **$200 – $1,000** | Typical squatter price for a non-trending name | Accept, or counter at ~50–70% of their quote. Don't haggle hard — small dollar amounts aren't worth weeks of negotiation. |
| **$1,000 – $3,000** | They think it has some commercial potential | Counter at ~30–50% of their quote. If they refuse to drop below $2,000, decide whether the .com is worth that to you (probably not — `.ai` works fine). |
| **$3,000 – $10,000** | They've Googled "praxant" since you emailed and saw something that suggested commercial value | Walk away unless you have a specific business reason to want `.com`. |
| **$10,000+** | They sense desperation or have positioned this as a "premium" name | Walk away. Politely decline. Use `praxant.ai`. |
| **"Not for sale at any price"** | Stealth entity building something | Walk away. Pay attention — they may emerge as a competitor. Track their domain via Visualping or similar quarterly. |

### 2.3 They reply but don't quote a number ("how much are you offering?")

**Do not give them a number.** Reply once with:

```
Hi,

Appreciate you getting back to me. Since I'm new to this and not
sure of fair value, could you let me know what you'd consider?
That way I can decide if it makes sense for my project.

Thanks,
Wellington
```

If they refuse to quote first, walk away. They're trying to extract your maximum.

### 2.4 They counter your counter

Negotiate once, then accept or walk. Long back-and-forth signals high willingness to pay and the price will keep climbing.

**Rule of thumb:** Final price = roughly 50–70% of their initial quote, settled in 1–2 counter-rounds.

## Step 3 — Use escrow for payment (NEVER wire directly)

**Domain transfer scams are common.** Common patterns:
- Seller takes payment, never transfers
- Seller transfers to a different registrar making transfer harder for you
- Fake seller intercepts the conversation and impersonates the real owner

### 3.1 Use Escrow.com

Industry-standard service. Costs ~$25 for transactions under $5,000.

Process:
1. Go to https://www.escrow.com
2. Create an "Acquisition" transaction:
   - Type: **Domain Name**
   - Description: **praxant.com**
   - Amount: **agreed price**
   - Buyer: **you**
   - Seller: **registrant's email**
3. Both parties create accounts and verify identity
4. You deposit the funds with Escrow.com (held by them, not given to seller yet)
5. Seller initiates the domain transfer (Step 4)
6. Once transfer completes and you confirm, Escrow.com releases funds to seller
7. If transfer fails or seller doesn't deliver, your money is refunded

### 3.2 Alternative: Sedo or Dan.com escrow

Both are domain marketplaces with built-in escrow. Slightly more expensive but tailored to domain transactions:
- Sedo: https://www.sedo.com
- Dan.com: https://dan.com

Use these if Escrow.com is unavailable in your or the seller's country.

### 3.3 Red flags during payment setup

| Red flag | Action |
|---|---|
| Seller refuses to use any escrow service | **Walk away.** Legitimate sellers welcome escrow because it protects them too. |
| Seller insists on cryptocurrency | **Walk away.** Untraceable, no recourse. |
| Seller asks for payment via wire / Western Union / gift cards | **Walk away.** These are scammer signals. |
| Seller's email changes mid-conversation | **Walk away.** Likely an interception scam. |
| Seller pushes hard for urgency ("must close today") | **Slow down.** Urgency is a manipulation tactic. |

## Step 4 — Transfer the domain

### 4.1 Receive the EPP/auth code from seller

The seller will provide a **transfer authorization code** (also called EPP code, auth code, or transfer secret). This is required to move the domain from their registrar to yours.

### 4.2 Initiate transfer at your registrar

Recommended registrars (in order of preference for OSS projects):
- **Cloudflare Registrar** — at-cost pricing, no markup, free WHOIS privacy
- **Porkbun** — cheap, good UI, free WHOIS privacy
- **Namecheap** — common, reliable
- **Gandi** — same registrar the domain is on; transfer is internal (faster but doesn't decouple from seller's account)

Process at your chosen registrar:
1. Go to their domain transfer page
2. Enter `praxant.com`
3. Enter the EPP/auth code from seller
4. Pay the transfer fee (usually 1 year of registration, ~$10-15)
5. Receive transfer confirmation email

### 4.3 Wait for transfer to complete

ICANN-mandated transfer process takes **5-7 days**. During this period:
- You'll get an email asking to confirm the transfer (click the link)
- The seller's registrar may send them an email to "approve" or "deny" — they should approve
- DNS continues to point to seller's nameservers until you change them post-transfer

### 4.4 After transfer completes

1. **Update DNS** at your new registrar to point `praxant.com` to wherever you want (likely the same as `praxant.ai` — your project landing page)
2. **Set up redirect** if you want `praxant.com → praxant.ai` (or vice versa)
3. **Enable WHOIS privacy** (free at Cloudflare/Porkbun)
4. **Lock the domain** to prevent unauthorized transfers
5. **Release Escrow.com funds** to the seller (your final action)

### 4.5 Update the project charter

Edit `project-specs.md`:
- Change `**Domain:** praxant.ai (reserved 2026-04-27)` to add `+ praxant.com (acquired YYYY-MM-DD)`
- Add a decision log entry

## What to do if any step fails

### If pre-flight check shows praxant.com is an active business

Stop the acquisition path. Reconsider whether the project name "Praxant" is safe to use:
- If their business is in AI / software / agents → real risk; consider name change
- If their business is unrelated (e.g., consulting, construction, food) → low conflict risk; proceed with `praxant.ai` and ignore the `.com`

### If seller responds with $5k+ quote

Walk away politely:

```
Hi,

Appreciate the response. That's outside what I can justify for
a personal project. Best of luck with the domain.

Thanks,
Wellington
```

This leaves the door open for them to come back with a lower price in 6-12 months if they don't sell to anyone else.

### If seller goes silent mid-negotiation

Send one polite check-in after 1 week, then move on:

```
Hi,

Just checking in on the praxant.com inquiry. If you've decided
not to sell, no problem — just wanted to confirm so I can move on.

Thanks,
Wellington
```

If still no reply after 2 more weeks → done. Move on.

### If domain transfer fails after escrow payment

Contact Escrow.com support. They have established processes for refunding funds when domain transfers don't complete. This is exactly why you used escrow.

### If the .com remains unattainable

**This is fine.** The Praxant project succeeds entirely on `praxant.ai`. Many successful modern OSS projects don't have `.com`:
- Bun → bun.sh
- Astral / Ruff / uv → astral.sh
- Temporal → temporal.io
- Elysia → elysia.run

The `.com` is a nice-to-have, not a requirement.

## Alternative monitoring path (if you don't acquire)

If you decide not to pursue acquisition, set up monitoring instead:

### Monitor the domain quarterly

- **Visualping** (https://visualping.io) — free tier, monitors any URL for changes; alerts you if praxant.com starts hosting content
- **Manual check** — calendar reminder every 3 months to visit `praxant.com`

### Monitor for trademark filings

- **USPTO TESS** (https://tmsearch.uspto.gov) — search "Praxant" quarterly to detect new filings
- **EUIPO TMview** (https://www.tmdn.org/tmview/) — same for EU

### Monitor for domain expiration

- **DomainTools / WhoIsXML / Whoisology** — alerts you when the domain expiration date passes without renewal
- praxant.com expires 2027-01-31; if not renewed, it goes to grace period (30 days) → redemption period (30 days, expensive to recover) → drop period (5 days) → public availability
- Set a calendar reminder for **2027-02-01** to recheck

### Defensive trademark filing for your project

Once Praxant launches publicly (Phase 5 of the project roadmap):
- File USPTO trademark for "Praxant" in classes 009 (computer software) and 042 (technology services) — ~$350 per class
- Establishes your priority date for use in commerce
- Protects against the praxant.com registrant later trying to oppose your trademark
- Doesn't get you the .com but gets you legal standing
